# Decisions — February 2026

## 2026-02-28

### Hierarchical Memory System Adopted
- **Decision:** Replace flat MEMORY.md with hierarchical index + detail files
- **Rationale:** Token efficiency, better context management, scalable
- **Structure:**
  - `MEMORY.md` — Lightweight index (~1.5k tokens)
  - `memory/people/` — Per-person detail files
  - `memory/projects/` — Per-project detail files
  - `memory/decisions/` — Monthly decision logs
  - `memory/context/` — Temporary active context
  - `memory/YYYY-MM-DD.md` — Daily session logs

### QMD Implementation
- **Decision:** Build QMD (Query-Memory-Data) system with SQLite + FTS5 + Vector
- **Files:** `qmd.js`, `qmd-vector.js`
- **API Endpoints:** `/api/qmd/search`, `/api/qmd/timeline`, `/api/qmd/entries`, etc.
