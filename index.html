APEX Memory System

Architecture

Dual-storage model: MEMORY.md is authoritative; SQLite is indexed replica for fast queries.

Why dual storage?
MEMORY.md is git-safe, human-readable, version-controlled. SQLite provides O(1) semantic search without API calls.

Specifications

Embeddings

all-MiniLM-L6-v2 (384-dim, local, no API key)

Performance

40‚Äì50ms per query (cached embeddings)

Storage

SQLite with embedding BLOB storage

Retrieval

Vector similarity + decay scoring

Deployment & Startup

Quick Start (All-in-One)

apex-startup.ps1 -Mode start

Launches Gateway + Nerve + Employee-Agent in parallel (separate terminals).

Auto-Start on Login

apex-startup.ps1 -Mode setup-autostart

Creates Windows Task Scheduler job APEX-System-Startup.

Known Limitations

  • Windows Cron: Unix-only. Using HEARTBEAT.md + Employee-Agent instead.
  • Nerve Auth: Disabled by default on 0.0.0.0. Run npm run setup to enable.
  • Unicode Encoding: Charmap warnings on stderr (non-fatal).
