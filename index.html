APEX Memory System (v1)

Architecture

Dual-storage model: MEMORY.md is authoritative; SQLite is indexed replica for fast queries.

Why dual storage?
MEMORY.md is git-safe, human-readable, version-controlled. SQLite provides O(1) semantic search without API calls.

Specs

  • Embeddings: all-MiniLM-L6-v2 (384-dim, local, no API key)
  • Retrieval: Phase 1 vector similarity + Phase 2 decay scoring
  • Performance: 40‚Äì50ms per query (cached embeddings)
  • Storage: SQLite at .memory.db
  • Portability: Windows/macOS/Linux, Python 3.10+, offline-first

API (apex_memory.py)

from apex_memory import APEXMemory

with APEXMemory() as m:
    # Semantic search
    results = m.recall("breaking news tech", max_results=5)
    
    # Log decision
    m.log_decision(
        title="Feature Decision",
        details="Chose X over Y because...",
        category="decision",
        tags=["architecture", "performance"]
    )
    
    # Log task
    m.log_task(
        name="Integrate service X",
        status="in_progress",
        tags=["integration"]
    )
    
    # Get session snapshot
    context = m.session_context()
    
    # Reindex (after external changes)
    m.reindex_memory()

Deployment & Startup

.\apex-startup.ps1 -Mode start

Launches Gateway + Nerve + Employee-Agent in parallel (separate terminals).

Creates Windows Task Scheduler job: APEX-System-Startup (runs at user logon).
